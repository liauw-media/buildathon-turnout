## Platform Requirements

**Development:**
- Node.js (modern version, LTS recommended)
- npm (installed with Node.js)

**Production:**
- Node.js runtime required (Next.js is not a static export in this build)
- Deployment targets: Vercel (native support), self-hosted Node servers
- File system access: Required for `data/commits.json` persistence

## Deployment & Hosting

**Configured for:**
- Vercel (native Next.js hosting, default choice)
- Self-hosted Node servers (any platform with Node.js)

**File Storage:**
- Local filesystem only: `data/` directory (relative to `process.cwd()` at runtime)
- Session data: HTTP-only secure cookies (handled by Next.js `cookies()` API)

**Stateless approach not applicable:**
- App relies on file system persistence; cannot scale horizontally without shared storage
- Suitable for single-instance deployments or with persistent volume mounts
